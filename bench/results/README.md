# Benchmark Outputs

Phase outputs are stored in this folder:

- `phase1_raw.csv` / `phase1_raw.json`:
  - generated by `bench/scripts/run_phase1_baselines.sh`.
- `phase4_benchmarks.csv` / `phase4_benchmarks.json`:
  - generated by `bench/scripts/run_phase4_benchmarks.sh`.
  - compares interpreter vs native `k run`, validates outputs, compares C baselines, and records timing metrics.
  - benchmark config (runs, warmup, compiler flags) is now stored in the JSON `config` section for reproducibility.
- `phase5_benchmarks.csv` / `phase5_benchmarks.json`:
  - generated by `bench/scripts/run_phase5_benchmarks.sh`.
  - validates container correctness and native-vs-baseline band gate for phase 5.
- `phase6_benchmarks.csv` / `phase6_benchmarks.json`:
  - generated by `bench/scripts/run_phase6_benchmarks.sh`.
  - reports first-run vs steady-run timings and cached plan/materialize counters for hetero container paths.
- `phase6_benchmarks_comparisons.csv`:
  - steady-state overhead and first-vs-steady speedup summary (promote/chunk/gather vs packed baseline).
- `phase7_benchmarks.csv` / `phase7_benchmarks.json`:
  - generated by `bench/scripts/run_phase7_benchmarks.sh`.
  - reports fused vs non-fused pipeline timings, checksum/plan correctness gates, and allocation counters.
- `phase8_benchmarks.csv` / `phase8_benchmarks.json`:
  - generated by `bench/scripts/run_phase8_benchmarks.sh`.
  - reports own/auto/blas matmul runtime modes, correctness gates, cache-hit counters, and C baseline ratios.
- `phase9_benchmarks.csv` / `phase9_benchmarks.json`:
  - generated by `bench/scripts/run_phase9_benchmarks.sh`.
  - reports spawn/join overhead, channel throughput-latency, parallel-for scaling curve, and par-reduce chunk sweep.
- `matmul_tuned_schedule.json`:
  - generated by `python3 tune/matmul_tuner.py`.
  - stores tuned matmul schedule parameters consumed by phase8 runtime.

Keep historical baselines by copying files with date suffixes.
